# Техническое задание для разработки системы управления персоналом

## Цель проекта

Разработать систему управления персоналом, которая позволит хранить, добавлять и отображать информацию о различных типах сотрудников образовательного учреждения.

## Требования к системе

### Классы и интерфейсы

1. Создать абстрактный класс `Person` с полями:
   - `name` (имя)
   - `age` (возраст)

2. Реализовать следующие классы, наследующие от `Person`:
   - `Staff` (сотрудник) с дополнительным полем `salary` (зарплата)
   - `Instructor` (преподаватель) с дополнительным полем `subject` (предмет)
   - `Student` (студент) с дополнительным полем `schoolName` (название школы)
   - `EmptyPerson` (пустой человек) для обработки ошибок

3. Создать интерфейс `Repository<T>` с методами:
   - `getSize()` - получение размера хранилища
   - `add(T person)` - добавление одного элемента
   - `add(T[] people)` - добавление массива элементов
   - `findById(int id)` - поиск элемента по идентификатору

4. Реализовать интерфейс `PersonRepository`, расширяющий `Repository<Person>`

5. Создать класс `ArrayBasedPersonRepository`, реализующий `PersonRepository` на основе массива

6. Создать интерфейс `DataPrinter<T>` с методом:
   - `displayAll(PersonRepository repository)` - отображение всех элементов хранилища

7. Реализовать класс `ConsoleDataPrinter`, реализующий `DataPrinter<Person>` для вывода данных в консоль

8. Создать класс `DataManager` для управления данными с использованием `PersonRepository` и `DataPrinter<Person>`

9. Реализовать класс `DataInitializer` для заполнения хранилища начальными данными

### Функциональные требования

1. Система должна позволять добавлять новых сотрудников, преподавателей и студентов
2. Система должна обеспечивать поиск персон по идентификатору
3. Система должна отображать информацию о всех в хранилище
4. Система должна поддерживать работу с различными типами хранилищ (массив, список)
5. Система должна обеспечивать вывод данных в консоль

### Нефункциональные требования

1. Использовать принципы объектно-ориентированного программирования
2. Обеспечить обработку ошибок при поиске несуществующих идентификаторов
3. Реализовать возможность расширения системы для работы с другими типами данных

## Ожидаемый результат

Разработанная система должна соответствовать предоставленному коду и включать все описанные классы, интерфейсы и функциональность. Система должна демонстрировать работу с различными типами персон, их хранение, поиск и отображение.
