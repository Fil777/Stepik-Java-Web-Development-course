# Техническое задание на доработку системы управления персоналом

## Цель доработки

Улучшить систему управления персоналом, добавив поддержку уникальных идентификаторов для объектов `Person` и обеспечив возможность поиска объектов по их идентификаторам в репозиториях.

## Требования к доработке

1. Добавить поле `id` типа `int` в класс `Person` для хранения уникального идентификатора каждого объекта.

2. Изменить конструктор класса `Person`, чтобы он принимал параметр `id` и инициализировал соответствующее поле.

3. Добавить метод `getId()` в класс `Person` для получения значения поля `id`.

4. Изменить реализацию метода `findById(int id)` в классе `ArrayBasedPersonRepository`:
   - Метод должен осуществлять поиск объекта `Person` по его идентификатору `id` в массиве `storage`.
   - Если объект с заданным `id` найден, метод должен возвращать его.
   - Если объект с заданным `id` не найден, метод должен возвращать объект `EmptyPerson`.

5. Изменить реализацию метода `findById(int id)` в классе `ListRepository`:
   - Метод должен осуществлять поиск объекта `Person` по его идентификатору `id` в списке `storage`.
   - Если объект с заданным `id` найден, метод должен возвращать его.
   - Если объект с заданным `id` не найден, метод должен возвращать объект `EmptyPerson`.

6. Обновить классы `Staff`, `Instructor` и `Student`:
   - Изменить конструкторы классов, чтобы они принимали параметр `id` и передавали его в конструктор родительского класса `Person`.

7. Обновить класс `DataInitializer`:
   - Добавить генерацию уникальных идентификаторов `id` для каждого создаваемого объекта `Person`.
   - Передавать сгенерированные идентификаторы `id` в конструкторы соответствующих классов при создании объектов.

## Ожидаемый результат

После выполнения доработки система управления персоналом должна иметь следующие улучшения:

1. Каждый объект `Person` должен иметь уникальный идентификатор `id`.

2. Репозитории `ArrayBasedPersonRepository` и `ListRepository` должны осуществлять поиск объектов `Person` по их идентификаторам `id`, а не по индексам.

3. Класс `DataInitializer` должен генерировать уникальные идентификаторы `id` для каждого создаваемого объекта `Person`.

4. Все остальные функциональности системы должны остаться без изменений.
