## Техническое задание на создание SmartDevice

### Цель
Создать класс SmartDevice, представляющий умное устройство освещения с базовыми функциями управления.

### Описание класса
Класс SmartDevice должен моделировать умное устройство освещения со следующими характеристиками и возможностями:

1. **Основные свойства:**
   - Уникальный идентификатор (id)
   - Название устройства (name)
   - Состояние включения/выключения (isOn)
   - Уровень яркости (brightness)
   - Текущий цвет (color)
   - Список поддерживаемых цветов (supportedColors)
   - Время последнего взаимодействия (lastInteraction)
   - Журнал активности (activityLog)
   - Таймер (timer)

2. **Конструктор:**
   - Принимает название устройства
   - Инициализирует все свойства начальными значениями

3. **Методы управления:**
   - Включение устройства (turnOn)
   - Выключение устройства (turnOff)
   - Установка яркости (setBrightness)
   - Установка цвета (setColor)
   - Добавление нового поддерживаемого цвета (addSupportedColor)
   - Установка таймера (setTimer)
   - Отмена таймера (cancelTimer)

4. **Методы получения информации:**
   - Получение списка поддерживаемых цветов (getSupportedColors)
   - Получение времени последнего взаимодействия (getLastInteraction)
   - Получение журнала активности (getActivityLog)
   - Получение идентификатора (getId)
   - Получение названия (getName)
   - Получение статуса устройства (getStatus)

5. **Дополнительные требования:**
   - Логирование всех действий в файл и в список activityLog
   - Обновление времени последнего взаимодействия при каждом действии
   - Ограничение яркости в диапазоне от 0 до 100
   - Проверка поддерживаемых цветов перед установкой

### Класс Timer 

```java
class Timer {
    private LocalDateTime endTime;

    public void set(int minutes) {
        this.endTime = LocalDateTime.now().plusMinutes(minutes);
    }

    public void cancel() {
        this.endTime = null;
    }

    public boolean isExpired() {
        return endTime != null && LocalDateTime.now().isAfter(endTime);
    }
}
```

# Инкапсуляция

Инкапсуляция - это принцип ООП, который заключается в **сокрытии внутреннего состояния и поведения** и предоставлении доступа к нему только через **открытые методы**. 

Это позволяет:
- Контролировать изменения 
- Защитить данные от несанкционированного доступа

Таким образом, инкапсуляция обеспечивает:
1. **Контроль** данных
2. **Безопасность** данных
