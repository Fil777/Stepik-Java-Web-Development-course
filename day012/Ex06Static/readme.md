# Различия между экземплярными и статическими методами в Java

В Java методы можно разделить на две основные категории: экземплярные (instance) методы и статические (static) методы. 

## Экземплярные методы

1. **Определение:** Экземплярные методы - это методы, которые принадлежат экземплярам класса. Они работают с конкретными экземплярами класса и имеют доступ к их состоянию (полям).

2. **Вызов:** Для вызова экземплярного метода необходимо сначала создать instance класса, а затем вызвать метод через ссылку на этот экземпляр.

3. **Доступ к полям:** Экземплярные методы имеют доступ к экземплярным переменным (полям) и могут изменять их значения.

4. **Ключевое слово `this`:** Внутри экземплярных методов можно использовать ключевое слово `this` для ссылки на текущий контекст.

Пример экземплярного метода:
```java
public class Person {
    private String name;

    public void sayHello() {
        System.out.println("Привет, меня зовут " + name); // или this.name 
    }
}

Person person = new Person();
person.sayHello();
```

## Статические методы и поля

1. **Определение:** Статические методы и поля принадлежат самому классу, а не его экземплярам. Они связаны с классом в целом и не имеют доступа к состоянию отдельных экземпляров.

2. **Вызов:** Статические методы и поля можно вызывать напрямую через имя класса, без необходимости создания экземпляра класса.

3. **Доступ к полям:** Статические методы и поля могут обращаться только к другим статическим переменным (полям) класса. Они не имеют доступа к экземплярным полям.

4. **Ключевое слово `this`:** Внутри статических методов нельзя использовать ключевое слово `this`, так как они не связаны с конкретным экземпляром класса.

Пример статического метода:
```java
public class MathUtils {
    public static int sum(int a, int b) {
        return a + b;
    }
}

int result = MathUtils.sum(5, 3);
```

## Основные отличия

1. **Доступ к состоянию:** Экземплярные методы имеют доступ к состоянию конкретного экземпляра класса (экземплярным переменным), тогда как статические методы могут обращаться только к статическим переменным класса.

2. **Вызов:** Экземплярные методы вызываются через экземпляр класса, а статические методы вызываются напрямую через имя класса.

3. **Использование `this`:** Экземплярные методы могут использовать ключевое слово `this` для ссылки на текущий контекст, а статические методы не используют `this`.

4. **Связь:** Экземплярные методы связаны с конкретными экземпляром класса и работают с их состоянием, а статические методы принадлежат классу в целом и не зависят от состояния отдельных объектов.

## Некоторые сценарии применения статических членов:

Вот некоторые распространённые сценарии использования ключевого слова `static`:

1. **Константы**: Объявление констант, которые являются неизменяемыми и общими для всех.  
Например:
   ```java
   public static final double PI = 3.14159;
   ```

2. **Методы-утилиты**: Создание статических методов, которые не требуют создания экземпляра класса и могут быть вызваны напрямую через имя класса. Например, методы в классе `Math`:
   ```java
   double result = Math.sqrt(25);
   ```

3. **Фабричные методы**: Использование статических методов для создания и возврата экземпляров класса. Это позволяет инкапсулировать логику создания объектов. Например:
   ```java
   public static Person createPerson(String name, int age) {
       return new Person(name, age);
   }
   ```

4. **Счётчики**: Хранение общего состояния или счётчика, разделяемого всеми экземплярами класса. Например, отслеживание количества созданных объектов:
   ```java
   private static int count = 0;
   
   public MyClass() {
       count++;
   }
   ```

5. **Блоки инициализации**: Использование статических блоков для инициализации статических переменных или выполнения определённых действий при загрузке класса. Например:
   ```java
   static {
       // Инициализация статических переменных
       // или выполнение других действий при загрузке класса
   }
   ```

6. **Вложенные классы**: Объявление вложенных классов как статических, чтобы они могли быть созданы без необходимости создания экземпляра внешнего класса. Например:
   ```java
   public class OuterClass {
       public static class NestedClass {
           // ...
       }
   }
   ```

7. **Singleton**: Реализация шаблона проектирования Singleton, гарантирующего существование только одного экземпляра класса. Например:
   ```java
   public class Singleton {
       private static Singleton instance = new Singleton();
       
       private Singleton() {}
       
       public static Singleton getInstance() {
           return instance;
       }
   }
   ```


*Я не рекомендую привыкать к использованию статики и, уж тем более, злоупотреблять ей - это одна из причин, по которой я очень мало уделяю внимания разбору static'и сейчас. В реальных приложениях её использую в ограниченном количестве сценариев и только тогда, когда есть точное понимание, зачем это делается.*
