# Принципы проектирования `SOLID`

Существует набор общепринятых принципов проектирования, которые позволяют определить соответствует ли программный код определению «хороший» или «плохой».
Этот набор состоит из 5 принципов. Термин `SOLID` является акронимом, образованным из первых букв названий отдельных принципов:
- [SRP](./solid/src/main/java/ru/ksergey/srp/readme.md) Single responsibility principle (принцип единственной ответственности).
- [OCP](./solid/src/main/java/ru/ksergey/ocp/readme.md) Open-closed principle (принцип открытости / закрытости).
- [LSP](./solid/src/main/java/ru/ksergey/lsp/readme.md) Liskov substitution principle (принцип подстановки Лисков).
- [ISP](./solid/src/main/java/ru/ksergey/isp/readme.md) Interface segregation principle (принцип разделения интерфейсов).
- [DIP](./solid/src/main/java/ru/ksergey/dip/readme.md) Dependency inversion principle (принцип инверсии зависимостей).

Следование каждому из этих принципов позволяет облегчить понимание и сопровождение кода. Кроме того, программный код, написанный согласно принципам `SOLID`, в большей мере соответствует и принципам объектно-ориентированного программирования.
