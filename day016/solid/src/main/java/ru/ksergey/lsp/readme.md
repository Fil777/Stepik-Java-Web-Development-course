# Liskov substitution principle

Принцип подстановки Лисков гласит, что объекты подклассов должны быть способны заменить объекты базового класса без изменения желаемых свойств программы. Иными словами, если у вас есть базовый класс и унаследованный от него подкласс, то подкласс должен быть совместим с базовым классом и не должен нарушать ожидаемое поведение программы.

### Пример

Плохой пример

```JAVA
public class Bird {
    public void fly() {
        System.out.println("The bird is flying.");
    }
}

public class Ostrich extends Bird {
    @Override
    public void fly() {
        throw new UnsupportedOperationException("The ostrich cannot fly.");
    }
}
```
Хороший пример

```JAVA
public class Bird {
    public void fly() {
        System.out.println("The bird is flying.");
    }
}

public class Ostrich extends Bird {
    @Override
    public void fly() {
        System.out.println("The ostrich cannot fly.");
    }
}
```

В хорошей реализации метод `Fly` в классе `Ostrich` переопределяется так, чтобы он корректно представлял поведение страуса, который не летает, но бегает. Таким образом, код, ожидающий объект типа Bird, может использовать объекты `Ostrich`, и они будут совместимы без нарушения LSP.

## Техническое задание: Моделирование движения птиц

### Описание:
Разработать систему для моделирования движения птиц и вычисления пройденного ими расстояния. Система должна поддерживать различные виды птиц, такие как обычные птицы и киви, и учитывать их особенности передвижения.

### Требования:

1. **Структура классов:**
   - Класс `Bird` для представления обычных птиц.
   - Класс `Kiwi`, наследующий от `Bird`, для представления киви.
   - Класс `Coordinates` для хранения координат птиц.
   - Класс `CalculatingDistance` для вычисления пройденного расстояния.

2. **Функциональность `Bird`:**
   - Хранение текущей позиции птицы.
   - Метод `move()` для перемещения птицы.
   - Метод `mark()` для отметки текущей позиции птицы.
   - Методы для получения скорости и пройденного расстояния.

3. **Функциональность `Kiwi`:**
   - Переопределение метода `move()` для учета особенностей передвижения киви.

4. **Функциональность `CalculatingDistance`:**
   - Вычисление пройденного расстояния за заданное время.

5. **Основная программа:**
   - Демонстрация движения обычной птицы и киви.
   - Вычисление пройденного расстояния для каждой птицы.

### Дополнительные требования:

1. Использовать наследование для моделирования различных видов птиц.
2. Обеспечить случайное направление движения птиц.
3. Выводить информацию о текущей позиции птицы при каждом перемещении.

### Примечания:

- Система должна быть расширяемой для добавления новых видов птиц с различными особенностями передвижения.
- Можно добавить дополнительные параметры, такие как скорость и направление движения, для более реалистичного моделирования.
- Следует учитывать принцип подстановки Лисков (LSP) при проектировании иерархии классов птиц.
