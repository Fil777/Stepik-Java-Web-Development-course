## Тренировка ArrayList

### 1. Список покупок

Реализуйте программу, которая позволяет пользователю создать список покупок. Программа должна запрашивать у пользователя продукты и добавлять их в ArrayList. Когда пользователь введет "конец", программа должна вывести весь список покупок.

### 2. Фильтрация чисел

Напишите программу, которая создает ArrayList из 20 случайных чисел от 1 до 100, а затем создает новый ArrayList, содержащий только четные числа из первого списка.

### 3. Удаление дубликатов

Создайте программу, которая принимает от пользователя список слов и удаляет все дубликаты, сохраняя порядок первого появления каждого слова.

### 4. Объединение списков

Напишите программу, которая создает два ArrayList с случайными числами, а затем объединяет их в третий список.

### 5. Реверс списка

Создайте программу, которая принимает от пользователя элементы списка, а затем создает новый ArrayList с элементами в обратном порядке.

### 6. Перемешивание списка

Напишите программу, которая создает ArrayList из 10 чисел от 1 до 10, а затем случайным образом перемешивает элементы этого списка. После перемешивания программа должна вывести новый порядок элементов.

Ознакомиться: [Тасование Фишера — Йетса](https://ru.wikipedia.org/wiki/Тасование_Фишера_—_Йетса#Оригинальный_метод_Фишера_—_Йетса)

### 7. Техническое задание на написание программы построения частотного словаря символов:

Разработать программу, которая будет анализировать заданный текст и строить частотный словарь символов, отображая количество вхождений каждого символа в тексте.

Функциональные требования:
1. Программа должна принимать на вход строку с текстом.
2. Программа должна подсчитывать частоту встречаемости каждого символа в тексте, включая буквы, цифры, знаки препинания и пробелы.
3. Программа должна хранить частоту символов в структуре данных HashMap, где ключом является символ, а значением - количество его вхождений в тексте.
4. Программа должна выводить построенный частотный словарь символов на экран в удобочитаемом формате.

Пример использования:

```text
Исходный текст: 
"Далеко-далеко за словесными горами в стране гласных и согласных живут рыбные тексты."

Частотный словарь: 
р=3
с=7
т=4
у=1
х=2
ы=6
а=7
б=1
в=3
г=3
д=2
е=6
ж=1
з=1
и=4
к=3
л=5
м=2
н=5
о=5
```

Построение частотного словаря символов может быть полезно в различных сценариях:

1. Анализ текста: Частотный словарь символов позволяет получить статистическую информацию о распределении символов в тексте. Это может быть полезно для лингвистического анализа, определения стиля автора или сравнения текстов.

2. Сжатие данных: Зная частоту встречаемости символов, можно разработать эффективные алгоритмы сжатия данных, такие как алгоритм Хаффмана. Символы с высокой частотой могут быть представлены более короткими кодами, что уменьшает размер сжатых данных.

3. Криптоанализ: Частотный анализ символов является одним из методов криптоанализа. Зная частоту встречаемости символов в зашифрованном тексте, можно попытаться расшифровать его, сопоставляя частоты символов с известными частотами в языке.

4. Генерация текста: Частотный словарь символов может быть использован для генерации реалистичного текста. Зная вероятности появления символов, можно создавать новые тексты, которые будут иметь похожие статистические свойства.

### 8. Техническое задание на разработку системы управления библиотекой

### Цель проекта
Разработать консольное приложение для управления небольшой библиотекой, позволяющее добавлять книги, регистрировать пользователей и отслеживать выдачу книг.

### Функциональные требования

1. **Управление книгами**
   - Добавление новых книг в библиотеку
   - Удаление книг из библиотеки
   - Поиск книг по названию

2. **Управление пользователями**
   - Регистрация новых пользователей
   - Выдача книг пользователям
   - Просмотр списка книг, взятых пользователем

3. **Пользовательский интерфейс**
   - Консольное меню с выбором действий
   - Ввод данных через консоль

### Технические требования

1. **Язык программирования**: Java
2. **Структуры данных**:
   - `ArrayList<String>` для хранения списка книг
   - `HashMap<String, ArrayList<String>>` для хранения пользователей и их книг
3. **Ввод/вывод**: Использование класса `Scanner` для ввода данных

### Детальное описание функций

1. **Добавление книги**
   - Запрос названия книги
   - Добавление введенной информации в список книг

2. **Удаление книги**
   - Запрос названия книги для удаления
   - Удаление книги из списка, если она найдена

3. **Поиск книги**
   - Запрос названия книги для поиска
   - Вывод всех книг, содержащих введенное название

4. **Добавление пользователя**
   - Запрос имени пользователя
   - Создание новой записи в `HashMap` для пользователя

5. **Выдача книги**
   - Запрос имени пользователя и названия книги
   - Проверка наличия книги в библиотеке
   - Добавление книги в список книг пользователя

6. **Просмотр книг пользователя**
   - Запрос имени пользователя
   - Вывод списка книг, взятых пользователем

### Дополнительные требования

1. Обработка некорректного ввода пользователя
2. Проверка наличия книги перед выдачей
3. Проверка существования пользователя перед выполнением операций

### Рекомендации по реализации

1. Использовать бесконечный цикл `while (true)` для работы главного меню
2. Применять обработку выбора пользователя
3. Использовать методы `add()`, `remove()`, `contains()` для работы с `ArrayList`
4. Применять методы `put()`, `get()`, `getOrDefault()` для работы с `HashMap`

### Дальнейшее развитие проекта

1. Реализация возврата книг пользователями
2. Добавление системы учета сроков выдачи книг
3. Сохранение данных в файл для долгосрочного хранения
4. Разделение кода на отдельные классы для улучшения структуры
